JanusTrace

A Python-based Requirement Traceability Framework for Critical Systems (e.g., DO-254)

Quick Start User Guide Developer Guide

JanusTrace bridges the gap between System Requirements and Source Implementation. It links defined requirements (from multi-file Excel or CSV databases) against codebase tags and generates interactive, JSON-backed HTML reports to verify 100% test coverage and flag orphans, omissions, and faulty configurations.

✨ Key Features

  • ► Flexible Tagging: Define custom requirement tag structures using Regex tokens via the Visual Configuration Builder.
  • ► Multi-Format & Multi-File Support: Seamlessly aggregate and map requirements from several `.xlsx` and `.csv` sheets in a single run.
  • ► Dynamic Language Parsing: Out-of-the-box support for VHDL, Verilog, SystemVerilog, C/C++, Python, Java, C#, Rust, MATLAB, and Ada. Extensible to any source file type via UI settings.
  • ► Interactive HTML Reports: Auto-generates sophisticated, standalone HTML matrices featuring sorting, searching, status filtering, and visual coverage statistics.

Quick Start

Installation

git clone https://github.com/your-repo/JanusTrace.git
cd JanusTrace
pip install -r requirements.txt

Running via GUI

We provide a modern, thread-safe Graphical User Interface built with customtkinter. It manages configurations, recent file states, and progress execution visually.

python main_gui.py

Running Executable

No Python? No problem. Simply download or build the standalone executable and launch it:

  • Windows: dist/JanusTrace.exe
  • Linux: dist/JanusTrace

User Guide

The GUI application is functionally divided into three main operational tabs:

1. Scan Project

This is the main execution environment. Here you define the targets for your traceability build:

  • Config File: Your custom YAML ruleset (e.g. config/default_rules.yaml). Validated securely on launch.
  • Requirements: The CSV/Excel files containing the ground truth descriptions and IDs (allows multiple inputs).
  • Source Directory: The root folder. JanusTrace will recursively find implementations inside configured language files.
  • Output Directory: Where the HTML and JSON reports will be dropped.

Clicking START SCAN initializes the backend TraceabilityEngine and provides real-time multi-stage feedback via the Progress Bar.

2. Configuration Builder

The heart of adaptability. Use this tab to dictate how exactly JanusTrace finds tags in your codebase.

  • Visual Builder: Snap together Fixed Texts (e.g., PROJ), Separators (-), and Variables (e.g., Digits (3)) to visually construct patterns without writing raw Regex.
  • Manual Regex: For experienced developers seeking complex constraints.
  • Tag Enclosement: Declare how tags wrapper inside strings (e.g., [ and ] to find [PROJ-REQ-001]).

3. Source Languages

JanusTrace strictly scans within language comments to avoid misidentifying code constants as requirement tags. By default, HDL (VHDL, Verilog) and Software (C++, Python) extensions are supported.

  • Enable/Disable complete language scanning blocks.
  • Edit or Add Custom Languages by specifying targeted file extensions (without dots, e.g., dart) and their relative line/block comment paradigms.
  • Conflict Resolution: Automatically identifies extension clashes between selected languages and warns the user prior to scanning.

Output Reports Analytics

Once generated, the interactive HTML report opens immediately. It provides:

  • Coverage Dashboard: Percentage tracking of implemented vs defined ground truth requirements.
  • Requirements Matrix: Clickable columns to sort by ID, Description, or Trace Status (OK, MISSING, INVALID).
  • Orphan Detection: Explicitly isolates tags found in code that do not correspond to any known Requirements document, saving hundreds of hours on deprecated hardware routines.

Artifact Cleanup Utility

Before committing to version control, you can quickly sanitize your workspace of generated reports, executables, and pycache using the built-in clean.py utility script.

# Preview what will be deleted without actually deleting
python clean.py --all --dry-run

# Wipe HTML/JSON reports and test outputs
python clean.py --reports

# Wipe PyInstaller build/ and dist/ folders
python clean.py --build

# Wipe all Python __pycache__ and test caches
python clean.py --cache

# Wipe everything (Reports, Builds, and Caches)
python clean.py --all

Developer Guide

System Architecture

JanusTrace is heavily modularized for Test-Driven Development (TDD) and separated into sub-packages within trace_framework/:

Package Responsibility
core.engine Contains the TraceabilityEngine responsible for diffing parsed docs against found code traces.
parsers Contains doc_parsers.py (CSV/Excel using pandas Base classes) and hdl_parsers.py (Regex comment matchers).
ui CustomTkinter View logic (gui_app.py), visual Config Builders, and the CLI execution wrappers.
utils Reporting abstractions (JSON & HTML template injections) and Configuration Validation pipelines.

Testing & CI/CD

TDD is mandated for this codebase. The framework provides a sprawling suite of 43 Pytest tests targeting deep engine logic, regex configuration, and edge-case validation, alongside 11 Standalone System Examples demonstrating real-world user flows.

1. Pytest Unit & Integration Tests (43 Tests)

  • test_config_validator.py: Ensures YAML configurations parse correctly and gracefully reject format errors.
  • test_engine.py: Validates core traceability, coverage statistics computation, and orphan detection logic.
  • test_cli_extensions.py: Checks that the CLI natively aligns with GUI file-extension parsing schemas.
  • test_report_gen.py: Asserts JSON and HTML artifact persistence, schema accuracy, and correct rendering.
  • test_config_helper.py: Verifies regex derivation from compound project requirement IDs.
# Run all 43 tests natively
python -m pytest tests/ -v

2. Standalone Examples (11 Scenarios)

The tests/ directory houses eleven complete user-environments combining nested code directories and CSV requirement DBs. These examples span typical scenarios (e.g., pure VHDL codebases) and edge-case exceptions (e.g., missing configurations, legacy IBM DOORS mapping differences format violations).

# Windows
cd tests && run_all.bat

# Linux / MacOS
cd tests && ./run_all.sh

GitHub Actions automatically triggers tests upon pull requests to the main branch via release.yml.

Building Executables

JanusTrace relies on PyInstaller for self-contained binaries, aggregating external libraries such as Pandas, Tkinter, and CustomTkinter automatically.

To compile explicitly:

pyinstaller --name JanusTrace \
  --onefile --windowed \
  --add-data "config;config" \
  --hidden-import="colorsys" \
  --hidden-import="trace_framework" \
  main_gui.py
